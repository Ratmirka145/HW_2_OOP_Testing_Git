# HW 2 OOP Testing Git

## Описание

Проект выполнен в рамках домашнего задания №2 по теме ООП, тестирование и Git.

В проекте реализована простая система для работы с рецептами, ингредиентами и списком покупок. Основной код находится в файле `recipes.py`.

В проекте есть четыре основных класса:

- `Ingredient` — ингредиент
- `Recipe` — обычный рецепт
- `DietaryRecipe` — диетический рецепт
- `ShoppingList` — список покупок

## Структура проекта

```text
HW_2_OOP_Testing_Git/
├── recipes.py
├── test_recipes.py
├── requirements.txt
├── .gitignore
└── README.md
```

## Классы проекта

### Ingredient

Класс `Ingredient` описывает один ингредиент.

Он хранит:

- название ингредиента
- количество
- единицу измерения

Количество должно быть положительным числом. Если передать ноль или отрицательное значение, программа вызовет ошибку `ValueError`.

### Recipe

Класс `Recipe` описывает рецепт.

Он хранит:

- название рецепта
- список ингредиентов

Класс умеет добавлять ингредиенты, объединять одинаковые ингредиенты и масштабировать рецепт на нужное количество порций.

### DietaryRecipe

Класс `DietaryRecipe` наследуется от класса `Recipe`.

Он дополнительно хранит тип питания, например:

- веган
- без сахара
- без глютена

При масштабировании такой рецепт остается объектом класса `DietaryRecipe`.

### ShoppingList

Класс `ShoppingList` нужен для создания списка покупок.

Он умеет:

- добавлять рецепт в список покупок
- удалять рецепт из списка покупок
- объединять одинаковые ингредиенты
- сортировать итоговый список по названию ингредиента
- объединять два списка покупок

## Установка

Сначала нужно склонировать репозиторий:

```bash
git clone https://github.com/Ratmirka145/HW_2_OOP_Testing_Git
```

Перейти в папку проекта:

```bash
cd HW_2_OOP_Testing_Git
```

Установить зависимости:

```bash
pip install -r requirements.txt
```
